# PRD: 칸반 보드 애플리케이션

## 1. 제품 개요

### 1.1 제품 비전
복수 사용자가 각자의 칸반 보드를 관리하고, 팀과 협업할 수 있는 직관적이고 확장 가능한 칸반 보드 웹 애플리케이션

### 1.2 제품 목표
- 설치 없이 브라우저에서 즉시 사용 가능한 칸반 보드 제공
- 드래그 앤 드롭을 통한 직관적인 작업 관리
- 빠른 로딩과 부드러운 사용자 경험
- **사용자별 독립적인 데이터 관리** (향후 인증 연동)
- 로컬 우선 동작, 인증 후 클라우드 동기화

### 1.3 타겟 사용자
- **개인 사용자**: 개인 프로젝트나 일상 작업을 관리하는 사용자
- **프리랜서**: 여러 프로젝트를 동시에 관리하는 독립 작업자
- **소규모 팀**: 간단한 작업 흐름을 공유하려는 3-5인 팀
- **학생**: 학습 계획이나 과제를 관리하는 학생

### 1.4 멀티 테넌시 전략
- **Phase 1 (MVP)**: 로컬 단일 사용자 (현재)
- **Phase 2**: LocalStorage + userId 구조로 복수 사용자 대비
- **Phase 3**: Supabase 인증 연동, 사용자별 데이터 분리
- **Phase 4**: 보드 공유 및 팀 협업 기능

## 2. 핵심 기능 요구사항

### 2.1 칸반 보드 구조
- **3개 기본 컬럼**: To-Do, In Progress, Done
- 각 컬럼은 독립적인 카드 컨테이너
- 컬럼별 카드 개수 실시간 표시
- 반응형 레이아웃 (데스크톱/모바일)
- **사용자별 독립된 보드** (userId 기반 데이터 격리)

### 2.2 카드 관리
#### 2.2.1 카드 생성
- 각 컬럼에서 "+ 카드 추가" 버튼 제공
- 모달 창을 통한 카드 제목 입력
- Enter 키로 빠른 추가 지원
- 필수 입력: 카드 제목 (1-200자)

#### 2.2.2 카드 이동
- 드래그 앤 드롭으로 컬럼 간 이동
- 시각적 피드백 (드래그 중 카드 스타일 변경)
- 드롭 가능 영역 하이라이트
- 부드러운 애니메이션 효과

#### 2.2.3 카드 삭제
- 각 카드에 "삭제" 버튼 제공
- 삭제 확인 다이얼로그 표시
- 삭제 후 카드 개수 자동 업데이트

### 2.3 사용자 경험
#### 2.3.1 시각적 피드백
- 드래그 중: 카드 투명도 50%, 5도 회전
- 드롭 가능 영역: 배경색 변경 및 점선 테두리
- 호버 효과: 카드 상승 및 그림자 강조
- 버튼 호버 시 색상 변경

#### 2.3.2 인터랙션
- 모달 창 외부 클릭으로 닫기
- ESC 키로 모달 닫기 (향후 추가)
- 빈 제목 입력 시 경고 메시지
- 삭제 시 확인 다이얼로그

## 3. 비기능 요구사항

### 3.1 성능
- 초기 로딩 시간: 1초 이내
- 드래그 앤 드롭 응답 시간: 50ms 이내
- 카드 추가/삭제 응답 시간: 100ms 이내
- 100개 카드까지 성능 저하 없이 동작

### 3.2 브라우저 호환성
- **필수 지원**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **선택 지원**: Opera, Brave
- 모바일 브라우저: iOS Safari 14+, Chrome Mobile 90+

### 3.3 접근성
- 키보드 네비게이션 (향후 개선)
- ARIA 레이블 (향후 개선)
- 색상 대비 WCAG 2.1 AA 준수

### 3.4 반응형 디자인
- **데스크톱** (1024px 이상): 3컬럼 수평 레이아웃
- **태블릿** (768-1023px): 3컬럼 축소 레이아웃
- **모바일** (767px 이하): 1컬럼 세로 레이아웃

## 4. 사용자 스토리

### 4.1 Epic 1: 카드 생성
```
As a 사용자
I want to 새로운 작업 카드를 쉽게 추가하고 싶다
So that 내 할 일을 빠르게 기록할 수 있다
```

**수용 기준**:
- [ ] 각 컬럼에 "+ 카드 추가" 버튼이 있다
- [ ] 버튼 클릭 시 모달 창이 열린다
- [ ] 제목을 입력하고 Enter 또는 "추가" 버튼으로 생성한다
- [ ] 빈 제목일 경우 경고 메시지가 표시된다
- [ ] 생성 후 카드가 해당 컬럼에 추가된다

### 4.2 Epic 2: 카드 이동
```
As a 사용자
I want to 카드를 드래그해서 다른 컬럼으로 옮기고 싶다
So that 작업 진행 상태를 시각적으로 업데이트할 수 있다
```

**수용 기준**:
- [ ] 카드를 마우스로 드래그할 수 있다
- [ ] 드래그 중 카드 스타일이 변경된다
- [ ] 드롭 가능한 컬럼이 하이라이트된다
- [ ] 컬럼에 드롭하면 카드가 이동한다
- [ ] 카드 개수가 자동으로 업데이트된다

### 4.3 Epic 3: 카드 삭제
```
As a 사용자
I want to 완료되거나 불필요한 카드를 삭제하고 싶다
So that 보드를 깔끔하게 유지할 수 있다
```

**수용 기준**:
- [ ] 각 카드에 "삭제" 버튼이 있다
- [ ] 삭제 시 확인 다이얼로그가 표시된다
- [ ] 확인 후 카드가 즉시 제거된다
- [ ] 카드 개수가 자동으로 업데이트된다

### 4.4 Epic 4: 데이터 지속성 (향후)
```
As a 사용자
I want to 브라우저를 닫아도 내 카드가 저장되기를 원한다
So that 다음에 다시 열었을 때 이어서 작업할 수 있다
```

**수용 기준**:
- [ ] LocalStorage에 카드 데이터 자동 저장 (userId별)
- [ ] 페이지 새로고침 시 데이터 복원
- [ ] 데이터 손실 방지
- [ ] 사용자별 데이터 격리

### 4.5 Epic 5: 사용자 인증 (Phase 3)
```
As a 사용자
I want to 로그인해서 내 데이터를 안전하게 관리하고 싶다
So that 다른 기기에서도 내 칸반 보드에 접근할 수 있다
```

**수용 기준**:
- [ ] Supabase Auth를 통한 이메일/소셜 로그인
- [ ] 로그인 후 사용자별 데이터 자동 로드
- [ ] 로그아웃 시 로컬 데이터 보호
- [ ] 세션 유지 및 자동 로그인

### 4.6 Epic 6: 다중 보드 관리 (Phase 3)
```
As a 사용자
I want to 여러 개의 보드를 만들어 프로젝트별로 관리하고 싶다
So that 개인/업무/학습 등을 분리하여 정리할 수 있다
```

**수용 기준**:
- [ ] 보드 생성/삭제 기능
- [ ] 보드 목록 조회
- [ ] 보드 간 전환
- [ ] 각 보드는 독립적인 카드 세트 보유

### 4.7 Epic 7: 보드 공유 및 협업 (Phase 4)
```
As a 사용자
I want to 내 보드를 팀원과 공유하고 싶다
So that 함께 작업을 관리하고 진행 상황을 공유할 수 있다
```

**수용 기준**:
- [ ] 이메일로 보드 멤버 초대
- [ ] 멤버별 권한 관리 (owner, editor, viewer)
- [ ] 실시간 카드 업데이트 동기화
- [ ] 멤버 활동 이력 표시

## 7. 제약사항

### 7.1 기술적 제약
- 백엔드 서버 없음 (프론트엔드 전용)
- 외부 라이브러리 최소화 (Vanilla JS)
- LocalStorage 용량 제한 (5-10MB)

### 7.2 비즈니스 제약
- 무료 오픈소스 프로젝트
- 단일 개발자/소규모 팀
- 빠른 개발 사이클 (1-2주)

### 7.3 사용자 제약
- **Phase 1-2**: 인터넷 연결 불필요 (로컬 동작)
- **Phase 3+**: 인증 및 동기화 시 인터넷 필요
- **Phase 1-2**: 계정 생성 불필요 (익명 사용)
- **Phase 3+**: Supabase 계정 필요 (선택)
- 데이터는 로컬 우선 저장, 인증 후 클라우드 백업
